// Hidden Chef Portal visibility state handler in App.tsx
const [showChefButton, setShowChefButton] = useState(() => {
  try {
    if (typeof window !== "undefined") {
      const params = new URLSearchParams(window.location.search);
      
      if (params.has("chef") || params.has("admin") || params.has("owner")) {
        // Returns true dynamically to show the admin lock button temporarily
        return true;
      }
      
      // Fallback to checking local storage if logo gesture was clicked 5 times
      return localStorage.getItem("shiva_refresh_show_chef_button") === "true";
    }
  } catch (e) {
    console.error(e);
  }
  return false;
});

Database & Storage Design

Serverless LocalStorage Schemas

🌿 Shiva Refresh
shiva_refresh_show_chef_button Boolean
Determines whether the administrative entrance lock icon remains visible on the header. Set via the 5-click logo gesture.
shiva_refresh_products JSON Array
Holds custom added, edited, or deleted menu products. Hydrates the active state of the store menu catalog.
shiva_refresh_orders JSON Array
Caches checkout orders locally with unique Order ID streams (e.g. `SR-7430`), customer addresses, times, and contents.
shiva_refresh_business JSON Object
Saves configuration settings: store open/closed indicator, chef description biography, and custom support dispatch numbers.

Technical Challenges & Solutions

Part 1: Order Formatting & Local Caching

🌿 Shiva Refresh
Challenge 1: Formatting Complex WhatsApp Message Payloads Completed
Problem: Customers ordering nested items (like customized bowls and sugar-free smoothies) require structured lists with prices, subtotals, and delivery coordinates that fit WhatsApp URI constraints.
Solution: Wrote a string template generator using `encodeURIComponent` with line-break character mapping (`\n`) and bold markdown formatting (`*`) to structure message summaries clearly on mobile displays.
Challenge 2: Serverless State Caching Completed
Problem: Standard React states reset on page refresh. Setting up external database hosting would add monthly costs for the business owner.
Solution: Built a data pipeline binding React hook lifecycle states directly to browser `LocalStorage`. Ensures the basket, menu updates, and orders remain synchronized locally for free.

Technical Challenges & Solutions

Part 2: Ingress Protection & URL Param Caching

🌿 Shiva Refresh
Challenge 3: Shared Device Admin Visibility Exposure Completed
Problem: If a chef logs in from a shared computer at the store, the Chef Portal entry button remains visible to other customers on that browser.
Solution: Designed a **Device Security** button inside settings that wipes the persistent `shiva_refresh_show_chef_button` key from local storage and redirects the page, securing administrative entry instantly.
Challenge 4: Caching of Temporary URL Query Flags Completed
Problem: Accessing the app with `?admin` saved the permission permanently to local storage, keeping the portal visible even after removing `?admin` from the URL.
Solution: Refactored the state initializer in `App.tsx` to read the search params dynamically during load. It displays the button temporarily without writing it to localStorage, so removing the parameter and reloading hides it.

Learnings, Limitations & Future Scope

Project Wrap-up and Evolution Roadmap

🌿 Shiva Refresh
Key Learnings
* React 19 State Synchronization
* Client-side persistence strategies
* Building responsive styling frameworks in Tailwind CSS
* Creating dynamic ingress triggers without backends
Limitations
* Caching is restricted to the browser: menu edits or order logs on one device are not visible on other devices.
* Offline modes require a browser cookie/local storage reset to clear space.
Future Scope
* Integration of external database (Supabase or Firebase) for multi-device sync.
